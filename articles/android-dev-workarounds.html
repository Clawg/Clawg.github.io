Android Dev workarounds

Input sytyling when the soft keyboard is visible

On some android devices (HTC 1 and Samsung Galaxy Tab 8inch) when the input get focus and the soft keyboard is visible the styling changes to default device styling.

This can be resolved with the following code:
-webkit-user-modify: read-write-plaintext-only; http://css-infos.net/property/-webkit-user-modify
-webkit-tap-highlight-color:rgba(0,0,0,0);
outline-style: none; Only apply this on focus of the element as it will cause the input to auto focused on othe devices if its the first element on the page
